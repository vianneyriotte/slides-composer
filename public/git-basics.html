01

git init

Initialise un nouveau dépôt Git dans le répertoire courant. Crée un dossier caché .git/ qui contient tout l'historique.

terminal
$ mkdir mon-projet $ cd mon-projet $ git init Initialized empty Git repository in /mon-projet/.git/

Au tout début d'un nouveau projet local

02

git clone

Copie un dépôt distant (GitHub, GitLab…) sur votre machine, avec tout son historique et ses branches.

terminal
$ git clone https://github.com/user/repo.git Cloning into 'repo'... done.

Pour récupérer un projet existant depuis un serveur distant

03

git status

Affiche l'état du répertoire de travail : fichiers modifiés, ajoutés à l'index, ou non suivis. Votre boussole au quotidien.

terminal
$ git status On branch main Changes not staged for commit:   modified: index.html Untracked files:   style.css

Avant chaque commit, pour savoir où vous en êtes

04

git add

Ajoute des fichiers à la zone de staging (index). C'est la sélection de ce qui sera inclus dans le prochain commit.

terminal
# Ajouter un fichier spécifique $ git add index.html # Ajouter tous les fichiers modifiés $ git add .

Pour préparer les changements avant de les valider

05

git commit

Enregistre un snapshot des fichiers staged dans l'historique. Chaque commit est un point de sauvegarde avec un message descriptif.

terminal
$ git commit -m "feat: ajout de la page d'accueil" [main a1b2c3d] feat: ajout de la page d'accueil 2 files changed, 48 insertions(+)

Après chaque modification logique — petits commits fréquents

06

git log

Affiche l'historique des commits. Utilisez les options pour un affichage compact ou graphique.

terminal
$ git log --oneline --graph * a1b2c3d feat: ajout page d'accueil * e4f5g6h init: structure du projet * i7j8k9l Initial commit

Pour explorer l'historique et comprendre l'évolution du projet

07

git push

Envoie vos commits locaux vers le dépôt distant. Partage votre travail avec l'équipe.

terminal
# Premier push (lier la branche) $ git push -u origin main # Pushs suivants $ git push

Après un ou plusieurs commits, pour synchroniser avec le remote

08

git pull

Récupère et fusionne les changements du dépôt distant. Combine git fetch + git merge en une seule commande.

terminal
$ git pull remote: Enumerating objects: 5, done. Updating a1b2c3d..m4n5o6p Fast-forward README.md | 12 +++++++++---

En début de session, pour récupérer le travail des collègues

09

git branch

Gère les branches : lister, créer ou supprimer. Les branches permettent de travailler en parallèle sans impacter le code principal.

terminal
$ git branch # lister * main   feature/login $ git branch feature/api # créer

Pour isoler une feature, un bugfix, ou une expérimentation

10

git switch

Change de branche. Remplace l'ancien git checkout pour la navigation entre branches (plus clair et moins d'erreurs).

terminal
# Changer de branche $ git switch feature/login # Créer + changer en une commande $ git switch -c feature/new # Équivalent ancien (fonctionne encore) $ git checkout feature/login

Pour naviguer entre les branches de travail

11

git merge

Fusionne une branche dans la branche courante. Intègre le travail parallèle dans le tronc commun.

terminal
# D'abord, se placer sur la branche cible $ git switch main # Puis fusionner la feature $ git merge feature/login Merge made by the 'ort' strategy.

Quand une feature est terminée et prête à rejoindre main
